Setup Guide

From zero to your operator
in about 15 minutes.

Before you start

Homebrew + developer tools install themselves in Step 2 — nothing to set up first.

STEP 0 / 6
01

Unzip the download

Find the aigent-os.zip file you downloaded. Double-click it to unzip — you'll get a folder called aigent-os. Put it somewhere you'll remember (Desktop or home folder works fine).

02

Install Homebrew

Open Terminal (press ⌘ Space, type Terminal, press Enter) and paste this in. It may ask for your Mac password.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

When it finishes: completely quit Terminal and open a brand-new window before continuing. Homebrew isn't on your PATH until a fresh shell loads it. (If Homebrew printed a "Next steps" box with an echo … >> ~/.zprofile line and an eval … line, paste those into the new window first.)

03

Connect your GitHub

In the new Terminal window, paste both lines. The first installs the GitHub CLI; the second links it to your account.

brew install gh gh auth login

When prompted, answer exactly this way: GitHub.com → HTTPS → Yes (authenticate Git) → Login with a web browser. It shows a one-time code — copy it, press Enter to open the browser, paste the code, and click Authorize. You only do this once.

04

Run the installer

Open Terminal (press ⌘ Space, type Terminal, press Enter). Navigate to the unzipped folder and run the installer:

cd ~/aigent-os bash install.sh

Replace ~/aigent-os with wherever you put the unzipped folder. The installer wires everything up — takes under a minute. If a Claude login prompt appears, sign in with your Pro or Max account.

05

Log in with Claude (only if asked)

Most people with Claude Code already set up are skipped straight to Step 6. If you are prompted, a browser opens — sign in with your Claude Pro or Max account. If anything later says authentication is missing, run:

claude setup-token
If it says login failed, it almost always means there's no active Pro/Max subscription on that account. Subscribe at claude.ai, then re-run: claude setup-token
06

Open your AIgent for the first time

Open a new Terminal window — this matters. The installer added the launcher to your PATH, which only loads in a fresh shell. Then type:

aigent

Your operator boots and runs its first-run setup. That's it — you're in.

Opening AIgent anytime

Three ways to launch it

  1. Spotlight — press ⌘ Space, type AIgent, press Enter.
  2. Applications folder — double-click AIgent.
  3. Any Terminal — type aigent.

The Spotlight shortcut and Applications app are created by the installer in Step 4.

Good to know

Three habits that make it work

  1. Open: start every session with /open — your AIgent loads your context and tells you exactly where to pick up.
  2. Close: when you're done, say close up — your AIgent banks the session so next time resumes clean. (Forgot? It recovers automatically.)
  3. Context counter (once): run /statusline in Claude Code and enable context usage. You'll see a live indicator as your session fills. When it gets heavy, close up, start fresh, and open again.

Troubleshooting

'brew' command not found after installing it
You need a fresh shell. Completely quit Terminal and open a new window, or run the eval "$(/opt/homebrew/bin/brew shellenv)" line that Homebrew printed at the end of its install.
'gh' command not found
Finish Step 3 (brew install gh) first.
"bash: install.sh: No such file or directory"
Your terminal isn't in the aigent-os folder. Run cd ~/aigent-os (or wherever you put it) first, then re-run bash install.sh.
Claude login failed
You need an active Claude Pro or Max subscription. Subscribe at claude.ai, then run: claude setup-token
'aigent' command not found / nothing happens
Open a brand-new Terminal window — the launcher is added to your PATH by the installer; new windows pick it up. If it's still missing: run source ~/.zshrc
I want to start over
Safe to re-run the Step 4 block. To wipe and redo cleanly: rm -rf ~/aigent then re-run Step 4.

Before you start

STEP 0 / 6
01

Unzip the download

Find the aigent-os.zip file you downloaded. Right-click it and choose Extract All — you'll get a folder called aigent-os. Put it somewhere you'll remember (Desktop or Documents works fine).

02

Install PowerShell 7

Open the Start menu, type PowerShell, open it, and paste this in:

winget install --id Microsoft.PowerShell --source winget --accept-package-agreements --accept-source-agreements

Then close that window. From now on, use PowerShell 7 (search pwsh in Start).

If it says "winget is not recognized", install "App Installer" from the Microsoft Store, then try again.

03

Find the installer

In the aigent-os folder you unzipped in Step 1, you'll find install.sh. Note the full path to this folder — you'll need it in Step 4.

04

Run the installer

Open PowerShell 7 and navigate to the unzipped folder, then run the installer:

cd $HOME\aigent-os bash install.sh

Replace $HOME\aigent-os with wherever you put the unzipped folder. The installer wires everything up — takes under a minute.

Windows users: bash requires Git for Windows (Git Bash). If bash is not recognized, install Git from git-scm.com first, then reopen PowerShell.
05

Log in with Claude

A browser window opens. Sign in with your Claude Pro or Max account and approve. That's the only login you need.

If it says login failed, it almost always means there's no active Pro/Max subscription on that account. Subscribe at claude.ai, then re-run: claude setup-token
06

You're in

An AIgent icon appears on your Desktop. Double-click it — your operator wakes up and runs /open. That's it.

Good to know

Three habits that make it work

  1. Open: start every session with /open — your AIgent loads your context and tells you exactly where to pick up.
  2. Close: when you're done, say close up — your AIgent banks the session so next time resumes clean. (Forgot? It recovers automatically.)
  3. Context counter (once): run /statusline in Claude Code and enable context usage. You'll see a live indicator as your session fills. When it gets heavy, close up, start fresh, and open again.

Troubleshooting

"pwsh" is not recognized
Finish Step 2 (install PowerShell 7), then close and reopen your terminal.
"winget" is not recognized
Install "App Installer" from the Microsoft Store, then retry Step 2.
"bash" is not recognized
You need Git for Windows. Download and install from git-scm.com (use all default options), then close and reopen PowerShell 7.
Claude login failed
You need an active Claude Pro or Max subscription. Subscribe at claude.ai, then run: claude setup-token
I want to start over
It's safe to re-run the Step 4 command anytime.
